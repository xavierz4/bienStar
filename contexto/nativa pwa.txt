Sobre el Punto 4: Limitaciones Offline de PWA vs App Nativa
El Problema Real para tu Caso
Tu app necesita funcionar sin internet en rancherías remotas de La Guajira. Esto es lo que cambia:

Funcionalidad	App Nativa (Expo)	PWA
Almacenamiento local grande	✅ WatermelonDB (ilimitado)	⚠️ IndexedDB (~50-100MB antes de pedir permiso)
Cámara y fotos	✅ Guardado permanente en disco	⚠️ Almacenamiento temporal (puede borrarse)
GPS	✅ Acceso completo	✅ Acceso completo
Sincronización en background	✅ Funciona con app cerrada	❌ Solo cuando la app está abierta
Instalación	App Store requerido	✅ Directo desde navegador
El Problema Crítico: Fotos y Almacenamiento
En la app nativa, cuando tomas una foto offline:

Se guarda permanentemente en FileSystem.documentDirectory
Sobrevive a cierres de app, reinicios del teléfono
Se sincroniza cuando hay internet
En PWA:

Las fotos van a Cache Storage o IndexedDB
El navegador puede borrarlas automáticamente si necesita espacio
Si el usuario borra datos del navegador, se pierde todo
La Solución: Persistent Storage API
Existe una API moderna que resuelve este problema:

// Solicitar almacenamiento persistente (no se borrará automáticamente)
if (navigator.storage && navigator.storage.persist) {
  const isPersisted = await navigator.storage.persist();
  console.log(`Almacenamiento persistente: ${isPersisted ? 'Sí' : 'No'}`);
}

Condiciones:

Limitación Aceptable Restante
La única limitación real es:

Si el usuario desinstala la PWA o borra datos del navegador = pierde datos no sincronizados
Mitigación: Mostrar alertas claras cuando hay datos pendientes de sincronizar.
